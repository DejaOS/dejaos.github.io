# 2026

## 07-20

### 更便捷地更换设备启动背景

此前，更换**设备**启动背景（上电 / 重启时显示的图片）通常需要重新制作或刷入固件，流程不便，也往往依赖厂商协助。

经过进一步研究，我们找到了更简单的方式：提供**对应机型的 DejaOS 升级应用**。将您想要的图片补充到我们提供的应用中，在设备上执行该应用即可更换启动背景，一般无需整包刷固件。

注意：

1. **不同设备的升级应用不一样**，需由我们按机型单独提供。该能力后续会集成到[系统管理应用](/docs/basics/managerapp)中，**暂时不会发布到 GitHub**。
2. **用户提供的图片分辨率必须与设备屏幕一致**。各机型分辨率可在 `dxDriver.js` 中查阅。
3. **出厂预装**不是少量订单的默认服务。若有**大批量**设备，可提供预装服务；否则可能需要额外收费，请联系 DejaOS 团队确认。

（应用程序启动界面背景仍是普通的 `dxui.Image` 资源，继续可通过更新工程资源替换。）

## 06-12

### DejaOS CLI 首次发布

新增独立的 **DejaOS CLI** 命令行开发工具。开发者无需安装 VSCode 插件，即可在终端中完成组件安装、项目构建、USB 设备连接、增量或完整代码同步、应用启动停止、设备重启和实时日志查看。

- 支持 `dejaos install`、`dejaos build`、`dejaos sync`、`dejaos sync --all`、`dejaos run` 和 `dejaos logs` 等命令；
- 提供 `dejaos edit` 终端项目配置编辑器，用于管理设备型号、SDK 版本和组件依赖；
- 支持通过 npm 全局安装：`npm install -g dejaos-cli`；
- 适用于终端开发、自动化脚本以及 AI 辅助编程场景。

详细使用方法请参阅 [DejaOS CLI](/docs/basics/cli) 文档。

## 04-13

### 引入 SDK 版本概念（2.0 / 3.0）

为支持可视对讲能力重构，我们将以设备摄像头为核心的能力链路进行了组件化拆分，并严格区分 `capture`、`ivcore`、`display`、`facial`、`intercom` 等模块，使同一路摄像头图像可以并行用于对讲、显示、识别等多种业务场景。由于新旧组件在架构与接口层面存在差异，版本之间不再兼容。  
为便于版本管理与项目选型，现正式引入 **SDK 版本**概念：**SDK 2.0** 包含旧组件体系下的全部版本，**SDK 3.0** 包含新组件体系下的全部版本。

## 04-02

### 人脸识别设备支持可视对讲功能

人脸识别设备现已支持可视对讲，访客可在门口设备发起呼叫，用户可通过室内机或手机 App 接听，实现实时视频与双向语音通话。在通话中确认来访身份后，可在同一流程内完成远程开门操作，并支持服务端私有部署。  
详细架构与通话流程请参阅[可视对讲系统概述](https://dejaos.com/docs/videointercom/overview/)文档。

## 01-10

### AI 生成的智能储物柜应用发布

我们很高兴宣布发布一个功能完整的**智能储物柜系统** —— 这是一个完全通过 AI 辅助开发生成的应用！

#### 亮点

- **100% AI 生成**：这个完整的嵌入式应用通过与 AI 助手进行 100+ 次对话创建完成，展示了 AI 在全栈嵌入式开发中的强大能力。
- **生产就绪**：与示例不同，这是一个功能完整的应用程序，只需少量自定义即可部署上线。
- **离线运行**：当前发布的是离线版本，完全独立运行，无需任何网络连接。

#### 主要功能

- **用户功能**：6 位密码存物、取物、实时显示可用储物柜数量
- **管理功能**：柜组管理、批量开柜控制、存取记录、时间设置、密码管理
- **多 Worker 架构**：UI、业务逻辑和硬件通信分离，实现最佳性能
- **硬件集成**：RS-485 与锁控板通信、PWM 蜂鸣器反馈

#### 即将推出

支持云端连接和**人脸识别**功能的**在线版本**将在后续发布。

详细文档和源代码请访问我们的[智能储物柜系统（离线版）](/apps/solutions/smart_locker_offline)文档。
