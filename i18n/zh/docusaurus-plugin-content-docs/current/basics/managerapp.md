# 系统管理应用

## 概述

系统管理应用（System Manager App）是 dejaOS 系统中用于执行底层维护任务的特殊应用。它主要用于应用程序安装、系统信息查询以及系统运行模式的切换。

随着系统版本的演进，系统管理应用也经历了从无界面的 PC 工具管理模式到有界面的设备端管理模式的升级。当前推荐版本为 **2.1.5**，在 **2.1.0** 基础上做了体验优化，并增加了更清晰的退出反馈、异常时的被动进入，以及面向生产环境的应用运行日志下载能力。

## 1. 新版本(2.1.0+)：设备端管理模式

新版本提供了可视化的系统管理应用，支持直接在设备屏幕上进行管理和配置，同时也支持通过局域网进行远程管理。

### 1.1 进入方式

设备启动时，会短暂显示灰色 / 过渡背景（约 2 秒）。在此期间，**长按屏幕任意位置**即可进入系统管理应用。注意，**一定是看到该背景之后才开始长按屏幕，而不是一开机就开始长按**。

详细步骤：

**步骤 1：设备上电，进入启动界面**

设备通电后，屏幕首先显示启动画面，提示「正在启动中」。

<img src="/img/sysapp_v215_boot.jpg" alt="启动界面" width="50%" />

**步骤 2：等待进入过渡界面，并及时长按屏幕**

数秒后自动进入过渡界面（浅色空白屏）。必须在此界面**长按屏幕**进入管理入口；若长时间不操作，请重启设备后重新操作。

<img src="/img/sysapp_v215_entry.jpg" alt="过渡界面（此时长按屏幕）" width="50%" />

**步骤 3：输入管理密码**

长按后进入 DejaOS Manager 密码页，输入管理员密码（默认 `314159`），点击 **CONFIRM** 确认。登录后可在系统配置中修改此密码。

<img src="/img/sysapp_v215_password.jpg" alt="密码输入界面" width="50%" />

**步骤 4：进入管理菜单并连接网络**

进入 DejaOS Manager 主菜单后，插入网线（或连接 Wi-Fi），相应配置后，点击 **Connect** 按钮，等待半分钟左右联网过程，回到首页。屏幕下方会显示设备地址，例如：

```
The web portal is now available at: http://192.168.10.113:8080
```

记下其中的设备 IP（实际以屏幕显示为准）。之后可在设备上操作各功能菜单，或在同一局域网的电脑上打开 Web 控制台。

<img src="/img/sysapp_v215_main.jpg" alt="DejaOS Manager（下方显示设备 IP 与端口）" width="50%" />

### 1.2 主要功能界面

主界面包含四大核心功能模块：

1.  **网络配置**：配置 Wi-Fi 或以太网连接。
2.  **系统信息**：查看 DejaOS 系统相关版本和已安装应用的版本、大小等基本信息。
3.  **系统配置**：修改系统密码、切换系统运行模式（参考 [系统模式](./mode.md)）。
4.  **安装应用**：支持通过网络下载或本地导入方式安装 DPK 应用包。

![主界面](/img/sysapp_main.png)

### 1.3 系统配置与模式切换

在系统配置页面，您可以直观地修改系统运行模式（开发、生产、测试；详见 [系统模式](./mode.md)）以及更改管理员密码。

![系统配置](/img/sysapp_config.png)

### 1.4 远程 Web 管理

除了在设备屏幕上操作外，系统管理应用还内置了一个 Web 服务器。您可以让电脑和设备连接在同一局域网下，通过电脑浏览器访问设备的 IP 地址，进入 Web 控制台进行远程管理。这种方式更为灵活简单。

![Web 管理界面](/img/sysapp_web.png)

通过 Web 管理页面，也可以实现应用包的升级和安装、运行模式的修改等操作。

### 1.5 2.1.5 版本增强

**2.1.5** 基于 **2.1.0**，保留原有进入方式、密码与核心功能模块，并增加以下改进。

#### 退出应用时界面变化更明显

此前退出应用后，屏幕往往会继续显示该应用的**最后一个页面**，容易让用户误以为应用仍在运行。

2.1.5 在退出应用时会切换到明确的状态页（例如生产模式提示页），便于确认上一应用已退出。

<img src="/img/sysapp_v215_prod_timeout.jpg" alt="退出应用后的明确界面" width="50%" />

#### 被动进入（看门狗熔断保护）

除常规的**灰屏长按**进入方式外，2.1.5 还支持**被动进入**：

若设备主应用因意外 Bug 或配置错误导致反复崩溃，看门狗检测到**连续重启达到 6 次**仍无法正常运行，系统将触发**熔断保护**。此时设备会**主动放弃启动主应用**，安全降级并**永久停留在系统管理应用**。

该机制可确保设备在极端异常下「不失联」，方便技术人员进入系统管理、提取日志并排查故障。

#### 应用运行日志下载

在**开发环境**下，运行日志可在 VS Code 中实时查看；在**生产环境**中则很难看到实时日志。从 **2.1.5** 起，可在进入系统管理应用后，通过 Web 控制台下载应用运行日志。

请先按 [1.1 进入方式](#11-进入方式) 进入系统管理，并记下主菜单上显示的设备 IP，然后：

**步骤 1：电脑浏览器访问管理后台**

1. 确认电脑与设备处于同一局域网
2. 浏览器访问 `http://<设备IP>:8080`（端口默认 **8080**）
3. 左侧菜单选择 **Maintenance & Logs**
4. 点击 **Download All Logs (.tar.gz)** 下载日志

<img src="/img/sysapp_v215_web_logs.png" alt="管理后台 · 点击 Download All Logs 下载" width="50%" />

**步骤 2：确认下载结果**

下载得到 `device_log.tar.gz`，解压后通常包含 `dejaos.log`。

<img src="/img/sysapp_v215_log_archive.png" alt="下载文件示例（device_log.tar.gz → dejaos.log）" width="50%" />

如果希望更实时地获取日志，也可以在自己开发的应用里，通过 `dxHttpClient` 等网络组件将日志文件上传到自有服务器。日志文件路径为 `/data/var/log/dejaos.log`。

## 2. 旧版本(2.0.0)：PC 工具管理模式

在旧版本的 dejaOS 系统中，系统管理功能通过一个被称为 **安全模式（Safe Mode）** 的特殊状态来实现。这种模式类似于 PC 电脑的 BIOS 模式，设备本身不提供图形交互界面，需要依赖 [PC 端的工具](https://github.com/DejaOS/DejaOS/blob/main/tools/tools.zip) 进行操作。

### 2.1 进入方式

设备启动时，有 2 秒的延迟窗口。在此期间，通过串口线连接 PC 工具即可进入此模式。

### 2.2 主要功能

- **安装 DPK 应用程序**：这是最核心的功能，用于将打包好的应用程序安装到设备上。
- **切换系统模式**：可以使用工具将设备切换到开发、生产或测试模式。
- **获取系统信息**：可以查看系统版本、硬件信息和已安装应用程序的基本信息。

![应用程序安装](/img/app_install2.png)

### 2.3 局限性

- 需要额外的硬件连接（串口转 USB 线缆）。
- 数据传输速率较低（受限于串口波特率），安装应用耗时较长。
