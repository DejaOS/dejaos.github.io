# DejaOS 版本差异说明

本文档详细介绍了 DejaOS 各个主要版本（2.1.0, 2.0.0, 1.0.0）的演进历程及关键差异。

> **注**：除了以下列出的功能性差异外，每个新版本通常都会包含稳定性的提升和已知 Bug 的修复。

---

## DejaOS 2.1.0 (最新推荐)

**DejaOS 2.1.0 是目前的最新推荐版本**。相比于 2.0.0，它重点提升了系统管理的便利性和开发体验。我们强烈建议所有 2.0.0 用户升级到此版本以获得最佳体验。

### 1. 系统管理方式：可视化升级

此版本引入了全新的 **System Manager App (系统管理应用)**，彻底替代了 2.0.0 中的安全模式。

- **2.0.0 (旧方式)**：依赖 **安全模式**，需通过 RS485 串口线连接，操作繁琐且传输慢。
- **2.1.0 (新方式)**：支持 **可视化屏幕操作** 和 **Web 远程管理**。无需特殊线缆，直接在设备屏幕或电脑浏览器上即可完成应用安装、模式切换等操作。

### 2. 开发体验：速度优化

- **优化了 USB 数据传输协议**：解决了 2.0.0 版本在开发模式下代码同步较慢的问题，大幅提升了调试效率。

---

## DejaOS 2.0.0

DejaOS 2.0.0 是一个里程碑版本，确立了现代 DejaOS 的应用架构标准。相比于 1.0.0，它在规范性和底层管理上做了重大改进。

> **升级建议**：虽然 2.0.0 版本功能完整，但由于开发体验和管理方式相对落后，建议尽快升级至 2.1.0 版本。

### 1. 维护机制：新增安全模式

- **1.0.0**：缺乏底层的应急管理机制。
- **2.0.0**：新增 **安全模式 (Safe Mode)**，支持通过串口工具进行系统维护和应用安装。

### 2. 应用规范：全面升级

- **安装包格式**：由 `.zip` 升级为 `.dpk` (DejaOS Package)。
  - _注：支持 VSCode 插件一键打包（默认全量），替代了 1.0.0 的手动压缩。_
- **目录结构**：引入了更规范的层级，源码路径增加了 `/app/code` 前缀（如 `/app/code/src/main.js`）。
- **OTA 路径**：升级包放置路径简化为 `/upgrades.zip`（1.0.0 为 `/app/data/upgrades/APP_1_0.zip`）。

---

## DejaOS 1.0.0 (已退出市场)

DejaOS 的早期初始版本，**目前基本已退出市场**，官方已停止主要维护。

- **主要特征**：
  - 无安全模式。
  - 使用普通的 `.zip` 包格式，需手动打包。
  - 应用目录结构较扁平（直接在根目录下）。
